# RPC-PMT 代码修改说明

## 1. 过滤无效过阈事件

- 改了什么：无效固定阈值时间改用 `NaN`，树增加 `threshold_valid`，所有阈值统计使用有限值判断。
- 为什么改：对应建议 1，原 `-1e6 ns` 哨兵会通过 `<100` 筛选。
- 影响范围：两个数据分析宏的事件树、阈值直方图、Profile 和 TA/TQ 修正。

## 2. 修复电荷积分边界

- 改了什么：左右积分都使用相邻采样点梯形积分，右循环要求 `i + 1 < size`。
- 为什么改：对应建议 2，原代码可访问 `signal[size]`。
- 影响范围：两个数据分析宏的 RPC/PMT 电荷结果。

## 3. 校验 CSV 波形输入

- 改了什么：要求至少三个等长采样点且时间严格递增，计算入口增加防御检查。
- 为什么改：对应建议 5，原代码可能在空或短数据上访问 `time[2]`。
- 影响范围：三个 ROOT 宏的 CSV 读取和波形预处理。

## 4. 独立计算固定阈值时间

- 改了什么：主宏独立从时间窗口起点搜索首次 10 mV 上升沿。
- 为什么改：对应建议 4，原搜索可能在找到固定阈值前提前退出。
- 影响范围：`RPC_PMT_DataAnalysis.C` 的固定阈值时间差。

## 5. 修正 PMT 高斯定时拟合

- 改了什么：右拟合边界补回平方根，检查拟合状态和参数；失败时保留插值定时。
- 为什么改：对应建议 6，原左右边界公式不一致且无失败处理。
- 影响范围：两个分析宏的 PMT 参考时间。

## 6. 分离时间修正训练与评估数据

- 改了什么：偶数索引事件拟合修正函数，奇数索引事件评估修正前后 sigma。
- 为什么改：对应建议 3，避免同一批事件拟合又评估造成乐观偏差。
- 影响范围：两个分析宏的 TA/TQ 时间分辨率。

## 7. 简化时间游走拟合模型

- 改了什么：十参数函数改为三参数 `p0+p1/sqrt(x)+p2/x`；检查有效 Profile bin 和拟合状态。
- 为什么改：对应建议 7，降低过拟合和边缘发散风险。
- 影响范围：两个分析宏的 TA/TQ 修正曲线。

## 8. 按实际 CSV 文件列表处理数据

- 改了什么：枚举数字命名 CSV 并按数值排序；单文件失败后继续。
- 为什么改：对应建议 8，消除连续编号假设和缺号截断。
- 影响范围：主分析宏和波形筛选器的文件遍历。

## 9. 为汇总树保存电压

- 改了什么：`tree_fitTAC_parameters` 增加并填充 `voltage` 分支。
- 为什么改：对应建议 9，使每条汇总记录可独立解释。
- 影响范围：两个分析宏的 ROOT 输出格式。

## 10. 管理逐事件 ROOT 拟合对象

- 改了什么：PMT 拟合函数使用唯一名称和指针拟合，参数提取后释放 `TF1`、`TGraph`。
- 为什么改：对应建议 10，避免长批次运行内存增长和 ROOT 名称冲突。
- 影响范围：两个分析宏的逐事件 PMT 高斯拟合。

## 新发现但未修改

- 百分比上升沿交点当前使用 `idx-1, idx`，按反向搜索语义更合理的包围点可能是 `idx, idx+1`。这会影响 rise time、CFD 和 FWHM，需要用真实波形确认后单独修改。
- 时间游走拟合质量检查的提前 `continue` 路径没有统一释放该电压已创建的临时 ROOT 对象。
- 仍有未使用变量和参数编译警告；本次没有将纯清理混入正确性提交。
- `Selected925` 的固定阈值从 -9 ns 开始向前搜索，噪声提前过阈的抗干扰策略仍需结合触发窗口确认。
