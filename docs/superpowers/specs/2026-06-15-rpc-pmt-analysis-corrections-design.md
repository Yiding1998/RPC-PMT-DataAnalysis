# RPC-PMT 分析代码修正设计

## 目标

按审查优先级修复三个 ROOT 宏中的错误结果、未定义行为和统计偏差。保留用户当前对
`RPC_PMT_DataAnalysis.C` 数据路径和输出名的未提交配置，不把它混入修复提交。

## 修改顺序

1. 显式标记并过滤无效过阈事件。
2. 修复电荷积分数组越界。
3. 拒绝无效、过短或时间非递增的 CSV 波形。
4. 独立搜索固定 10 mV 上升沿交点。
5. 修复 PMT 高斯拟合范围公式并增加失败回退。
6. 将时间游走拟合与分辨率评估拆分为训练集和验证集。
7. 简化时间游走模型并检查拟合质量。
8. 枚举并排序实际 CSV 文件，不假设编号连续。
9. 为时间修正汇总树增加电压分支。
10. 修复逐事件 ROOT 对象生命周期。

## 兼容性

保留三个宏的入口函数以及主要 ROOT 对象名称。新增有效性分支，错误事件不再进入统计。
修正后的分辨率只由验证集估计，因此不能与旧版同名结果直接视为相同估计量。

## 测试

每项先增加可复现检查，再修改代码。最终执行警告级语法编译、ROOT 宏加载、代表性数据检查和
`git diff --check`。实施中发现的范围外问题只记录，不擅自修改。
